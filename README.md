# XNH_WEBAPP_GIF-Conv

ブラウザだけで画像・動画・GIF を相互変換するシングルページ Web アプリです。  
サーバー不要で、`index.html` を開くだけで利用できます。

## 概要

- アプリ名: `Pic2GIF`
- 実装形態: 単一 HTML（React + Babel + Tailwind を CDN 読み込み）
- 動作場所: 完全クライアントサイド（ローカル実行 / GitHub Pages 配信可）
- 変換モード:
  - `画像/動画 -> GIF`
  - `GIF -> 動画 (WebM)`
  - `GIF -> 画像ZIP`

## 現状（2026-02-27 時点）

- 実装済み:
  - 3モード変換（画像/動画->GIF、GIF->動画、GIF->画像ZIP）
  - `ms` / `Hz` 双方向入力（`1〜240Hz`）とスライダー+数値入力UI
  - 出力横幅 `1〜4000px`、最大色数 `2〜256`、品質 `1〜10`
  - 進捗表示、キャンセル、プレビュー、ダウンロード
- 実装上の制約:
  - GIF は `10ms` 単位保存のため高Hz入力は近似値保存
  - 動画生成は `MediaRecorder`、GIFデコードは `ImageDecoder` / `gifuct-js` 依存
- 未対応 / 改善候補:
  - MP4 出力オプション
  - フレーム個別遅延編集
  - 動画抽出フレーム上限のUI設定

## 現在の仕様

### 1. 変換モード

- `画像/動画 -> GIF`
  - 入力: 画像複数ファイル、動画ファイル（ブラウザが読める形式）
  - 出力: `animation.gif`
- `GIF -> 動画 (WebM)`
  - 入力: GIF 1ファイル
  - 出力: `animation.webm`
- `GIF -> 画像ZIP`
  - 入力: GIF 1ファイル
  - 出力: `*-frames.zip`（連番 PNG + `frames.json`）

### 2. 入力とフレーム管理

- ドラッグ&ドロップ / クリック選択に対応
- `画像/動画 -> GIF` モード:
  - 画像を直接フレームとして追加
  - 動画はフレーム抽出して追加（過剰フレームを避ける上限あり）
  - フレームの左右移動・削除・全クリアに対応
- `GIF -> *` モード:
  - GIF をフレーム展開してプレビュー表示
  - 展開フレームごとの遅延(ms)を保持

### 3. フレームレート入力（ms / Hz 両対応）

- `Hz` 入力: `1`〜`240`（スライダー + 数値）
- `ms` 入力: `4.166...`〜`1000`（スライダー + 数値）
- 相互連動:
  - `ms = 1000 / Hz`
  - `Hz = 1000 / ms`
- 小数入力対応（Hz は小数2桁、ms は小数3桁表示）

### 4. GIF 出力設定

- ループ回数: `0`〜`10`（`0` は無限）
- 出力横幅: `1`〜`4000` px
- 品質: `1`〜`10`（`10` が高品質）
- 最大色数: `2`〜`256`

補足:
- 出力高さは 1枚目フレーム比率から自動計算
- レターボックスで中央配置（余白は白）
- `gif.js` 品質値変換: `gifQuality = max(1, round(21 - uiQuality * 2))`
- 色数制御は RGB 量子化で「上限色数」に近づける方式

### 5. GIF 仕様由来の制約（実装済み）

- GIF 保存遅延は `10ms` 単位に丸め
- 保存遅延:
  - `savedMs = max(10, round(inputMs / 10) * 10)`
  - `savedHz = 1000 / savedMs`
- 例:
  - `240Hz` 入力（約 `4.167ms`）でも保存は `10ms`（`100Hz` 相当）

### 6. 生成・進捗・キャンセル

- 変換中は進捗バー表示（`0`〜`100%`）
- 変換中キャンセル対応
- 完了後:
  - 出力プレビュー表示（GIF / 動画）
  - ファイルサイズ表示
  - ダウンロードボタンで保存

### 7. プレビュー

- 共通プレイヤー:
  - 再生 / 一時停止
  - 前 / 次フレーム
  - 最初 / 最後フレームジャンプ
- `GIF -> *` モードではフレーム固有遅延を反映

### 8. UI

- ライト / ダークモード切替
- システム配色を初期値として反映
- レスポンシブレイアウト（モバイル〜デスクトップ）

## 使用技術

- React `18.2.0`（CDN）
- ReactDOM `18.2.0`（CDN）
- Babel Standalone `7.23.5`（CDN）
- Tailwind CSS（CDN）
- gif.js `0.2.0`（GIF エンコード）

- JSZip `3.10.1`（ZIP 出力）
- Browser APIs:
  - `ImageDecoder`（利用可能環境では GIF デコード優先）
  - `MediaRecorder`（WebM 生成）

## 使用方法

### 画像/動画 -> GIF

1. モードを `画像/動画 -> GIF` にする
2. 画像または動画を追加
3. 必要ならフレーム順序を調整
4. `Hz/ms`、ループ、幅、品質、最大色数を調整
5. `GIF 生成` を押す
6. 完了後にダウンロード

### GIF -> 動画(WebM)

1. モードを `GIF -> 動画(WebM)` にする
2. GIF を追加
3. `動画生成 (WebM)` を押す
4. 完了後にダウンロード

### GIF -> 画像ZIP

1. モードを `GIF -> 画像ZIP` にする
2. GIF を追加
3. `画像ZIP生成` を押す
4. 完了後にダウンロード

## 実行時の注意点

- CDN 読み込みが必要なため、オフラインでは動作しません
- 非常に長い動画や高フレームレート指定は処理時間・メモリ消費が大きくなります
- GIF の時間分解能は `10ms` 単位のため、高 Hz 入力は保存時に近似されます
- `MediaRecorder` / `ImageDecoder` の対応状況はブラウザ依存です

## 技術メモ

### gif.js Worker の読み込み

```javascript
fetch('https://cdn.jsdelivr.net/npm/gif.js@0.2.0/dist/gif.worker.js')
  .then(r => r.blob())
  .then(blob => { workerUrlRef.current = URL.createObjectURL(blob); });
```

### GIF デコード方針

- まず `ImageDecoder` を試行
- 非対応/失敗時は `gifuct-js` にフォールバック

### メモリ管理

- 出力 Blob URL は再生成時に `URL.revokeObjectURL()` で解放
- worker Blob URL はアンマウント時に解放
