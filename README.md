# XNH_WEBAPP_GIF-Conv

ブラウザだけで複数画像を GIF アニメーションへ変換するシングルページ Web アプリです。  
サーバー不要で、`index.html` を開くだけで利用できます。

## 概要

- アプリ名: `Pic2GIF`
- 実装形態: 単一 HTML（React + Babel + Tailwind を CDN 読み込み）
- GIF 生成: `gif.js` を利用（Web Worker で非同期エンコード）
- 動作場所: 完全クライアントサイド（ローカル実行 / GitHub Pages 配信可）

## 現在の仕様

### 画像入力

- 入力方法:
  - ドラッグ&ドロップ
  - クリックしてファイル選択
- 複数ファイル同時追加に対応
- 画像 MIME タイプ（`image/*`）のみ処理
- 主な対応形式: PNG / JPG / WebP / GIF

### フレーム管理

- 追加した画像をフレームとして一覧表示
- サムネイルグリッドはレスポンシブ表示（`4`〜`10` 列）
- 各フレームに対して以下を実行可能:
  - 左右移動（順序変更）
  - 個別削除
- 全フレーム一括クリア
- 任意フレームをクリックしてプレビュー対象を変更

### プレビュー

- 選択フレーム表示
- 再生 / 一時停止
- 前 / 次フレーム移動
- 最初 / 最後フレームへジャンプ
- 再生速度は「フレーム遅延」設定値を使用

### GIF 出力設定

- フレーム遅延: `10`〜`3000` ms（既定 `100`）
- ループ回数: `0`〜`10`（既定 `0`、`0` は無限ループ）
- 出力幅: `64`〜`1200` px（既定 `480`）
- 品質（UI）: `1`〜`10`（既定 `7`）

補足:
- 出力高さは 1 枚目の画像アスペクト比から自動算出
- 画像ごとの縦横比が異なる場合はレターボックス（余白白色 `#ffffff`）で中央配置
- `gif.js` 品質値への変換式: `max(1, round(21 - uiQuality * 2))`

### 生成・ダウンロード

- 「GIF 生成」で非同期エンコード開始
- 進捗（`0`〜`100%`）を表示
- 生成中キャンセル可能
- 完了後:
  - GIF プレビュー表示
  - ファイルサイズ表示
  - `animation.gif` としてダウンロード可能

### UI / 表示

- ライト / ダークモード切替
- システムのダークモード設定を初期値として反映
- レスポンシブレイアウト（モバイル〜デスクトップ対応）

## 使用方法

1. `index.html` をブラウザで開く
2. 画像ファイルをドラッグ&ドロップ、またはクリックで複数選択
3. 必要に応じてフレーム順序を調整
4. 設定（遅延・ループ・幅・品質）を調整
5. 「GIF 生成」を押して完了まで待つ
6. 生成結果を確認し、「ダウンロード」で保存

## 実行時の注意点

- 初回ロード時、`gif.worker.js` を CDN から取得できないと生成できません
- オフライン環境では CDN 読み込みに失敗するため動作しません
- 非常に大きい画像・大量フレームはブラウザメモリ使用量が増えます

## 技術メモ

### gif.js の Web Worker 問題と対処

- `gif.js` は GIF エンコードに Web Worker を使用します
- Worker はクロスオリジン URL から直接ロードできないため、`fetch()` + Blob URL で回避しています

```javascript
fetch('https://cdn.jsdelivr.net/npm/gif.js@0.2.0/dist/gif.worker.js')
  .then(r => r.blob())
  .then(blob => { blobUrlRef.current = URL.createObjectURL(blob); });
```

### 出力サイズ計算

- 出力幅はユーザー設定値 `settings.width`
- 出力高さは 1 枚目フレームのアスペクト比から自動計算

```text
outH = floor(firstFrame.naturalHeight * outW / firstFrame.naturalWidth)
```

### 品質設定の変換

- UI スライダー: `1`〜`10`（`10` が高品質）
- gif.js `quality` パラメータ: `1`〜`20`（`1` が高品質）
- 変換式: `gifQuality = max(1, round(21 - uiQuality * 2))`

### メモリ管理

- 生成済み Blob URL は次回生成時に `URL.revokeObjectURL()` で解放
- Worker Blob URL はコンポーネントアンマウント時に解放

## これから実装する内容（実装予定）

### 1. 変換機能の拡張

- [ ] 画像に加えて、動画から GIF へ変換できる機能を追加
- [ ] GIF から動画へ変換できる機能を追加
- [ ] GIF から画像（フレーム書き出し）へ変換できる機能を追加

### 2. フレームレート入力の拡張

- [ ] フレームレートを `ms` と `Hz` の両方で入力できる UI を追加
- [ ] `ms` と `Hz` は相互連動（片方変更時にもう片方を自動更新）
- [ ] フレームレート入力は `1`〜`240` Hz の範囲でシームレスに変更可能
- [ ] 操作方式はスライダーバー + 数値入力に対応
- [ ] GIF 出力時は仕様上 `10ms` 単位へ丸めて保存（centisecond 単位）
- [ ] `240Hz` 入力時も GIF 保存は近似値へ変換（実効上限は概ね `100Hz` 相当）

### 3. 画質・解像度設定の拡張

- [ ] 画質設定に「最大色数」を追加（選択範囲: `2`〜`256` 色）
- [ ] 出力横幅（画素数）は `1`〜`4000` px の範囲でシームレスに変更可能
- [ ] 出力横幅の操作方式はスライダーバー + 数値入力に対応

### 4. 技術制約の明記（実装方針）

- [ ] GIF の時間解像度は `1/100` 秒（`10ms`）単位であることを UI と説明文に明記
- [ ] 高フレームレート設定は「入力値」と「GIF 保存値」が一致しない場合があることを明記
- [ ] ブラウザ再生環境によって体感再生速度が前後する可能性を注意書きに追記

### 5. 実装予定の詳細仕様（UI / バリデーション / 丸め）

#### 5-1. 変換モード

- `Image/Video -> GIF`
  - 入力: 画像複数ファイル、または動画ファイル（初期対応想定: MP4 / WebM）
  - 出力: `animation.gif`
- `GIF -> Video/Images`
  - 入力: GIF ファイル
  - 出力:
    - 動画: 初期は WebM を優先（ブラウザ互換性重視）
    - 画像: フレーム連番で書き出し（必要に応じて ZIP 化）

#### 5-2. フレームレート入力仕様

- 入力 UI:
  - Hz 入力: スライダー + 数値入力（`1`〜`240`）
  - ms 入力: スライダー + 数値入力（`4.1667`〜`1000` 相当）
- 相互変換:
  - `ms = 1000 / Hz`
  - `Hz = 1000 / ms`
- 表示値:
  - 小数を許可（例: Hz は小数 2 桁、ms は小数 3 桁まで）
  - フォーカスアウト時に規定桁へ正規化

#### 5-3. GIF 保存時の丸め仕様

- GIF の遅延値は `10ms` 単位へ丸める
- 保存用遅延値: `savedMs = max(10, round(inputMs / 10) * 10)`
- 保存用実効フレームレート: `savedHz = 1000 / savedMs`
- 例:
  - 入力 `240Hz`（`4.1667ms`） -> 保存 `10ms`（`100Hz` 相当）
  - 入力 `59.94Hz`（`16.683ms`） -> 保存 `20ms`（`50Hz` 相当）

#### 5-4. 画質・解像度入力仕様

- 最大色数:
  - 入力範囲: `2`〜`256`
  - UI: スライダー + 数値入力
  - GIF の量子化時に「上限色数」として適用
- 出力横幅:
  - 入力範囲: `1`〜`4000` px
  - UI: スライダー + 数値入力
  - 高さはアスペクト比維持で自動計算

#### 5-5. バリデーション方針

- 数値入力は `min/max` を超えた値をクランプ
- 空文字・非数値入力は直前の有効値へロールバック
- 保存前に最終検証を実施し、実際に保存される値（`savedMs`, `savedHz`, 色数, 解像度）を UI に表示
